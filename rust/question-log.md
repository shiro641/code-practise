### 2026-06-14：借用与引用 `&T`

问题：

- `other = &name` 有没有拿走 `name` 的所有权？
- `println!("{}", name)` 为什么可以？
- 借用的本质是不是地址的复制？地址是一种类型吗？它是不是实现了 `Copy` trait？
- 可以对引用取引用吗？
- 引用在底层到底保存的是什么？是原始值在栈上的地址吗？
- “指向”在底层是怎么实现的？

回答摘要：

- `other = &name` 没有拿走所有权，只是借用了 `name`。
- `name` 仍然拥有字符串 `"codex"`，所以后续可以继续使用 `name`。
- 可以把不可变引用粗略理解成复制了一份“指向原值的位置”。但 Rust 里的引用不只是裸地址，引用类型还带有借用规则和类型信息，例如 `&String`、`&str`。
- 地址本身不是日常 Rust 代码里直接操作的普通类型；引用 `&T` 是一种 Rust 类型。不可变引用 `&T` 通常是 `Copy`，复制引用不会复制被引用的数据。
- `&str` 更特殊一点，通常包含指针和长度；复制 `&str` 也是复制这份引用信息，不复制字符串内容。
- 可以对引用再取引用，例如 `let c = &b;`，如果 `b` 是 `&String`，那么 `c` 的类型可以理解为 `&&String`。这表示 `c` 引用的是引用 `b` 本身，而 `b` 再指向原始 `String`。Rust 很多场景会自动解引用，所以 `println!("{}", c)` 通常也能打印最终字符串。
- 普通 `&T` 底层通常保存被引用值的位置。是否是栈地址取决于被引用的值放在哪里：`let s = String::from("codex"); let r = &s;` 中，`r: &String` 指向栈上的 `String` 结构体；这个结构体内部再指向堆上的字符串数据。`&str` 通常直接包含字符串数据的位置和长度，如果来自 `s.as_str()`，它会指向堆上的字符数据而不是栈上的 `String` 结构体。
- “指向”底层通常由机器地址/指针值实现。`&T` 这类普通引用通常可以看作保存一个地址；CPU 通过这个地址去内存中读取目标值。Rust 额外做的是在编译期检查这个地址对应的借用是否安全，例如引用不能比被引用的值活得更久、不能违反读写规则。运行时通常不会给普通引用额外附带所有权标记。

后续融入方式：

- 后续讲引用时，用“复制引用，不复制数据”作为核心直觉。
- 后续讲 `&str` 时补充它不是单纯一个地址，而是指针加长度。
- 后续如遇到 `&&T`，用“引用的引用”解释，不把它和所有权转移混在一起。
- 后续讲 `String`、`&String`、`&str` 时，用“栈上的 String 结构体”和“堆上的字符串内容”区分引用指向的位置。
- 后续如提到底层，把“指针值是运行时数据，借用规则主要是编译期约束”作为核心区分。

### 2026-06-13：ownership 与 move 的基本规则

问题：

- `println` 后面为什么要加 `!`？
- `"{}, other"` / `"{}", other` 是什么意思？
- Rust 里面有什么类型？
- 以上类型里，只有 `String` 会发生所有权转移吗？
- `&str` 有 Copy 吗？
- `let a = "hello"; let b = a; b = "world"; println!(a)` 是否会打印 `world`？`let a = 1; let b = a; b = 2; println!(a)` 是否仍然为 `1`？
- Rust 里有没有类似 C 语言通过指针修改变量，或 JS 通过地址修改变量的语法？
- 为什么 `{integer}` 报 `no method named copy`？`i32` 没实现 Copy 吗？

回答摘要：

- 正确写法是 `println!`，不是普通函数 `println`。
- `println!` 后面有 `!`，表示它是一个宏。宏可以在编译期展开代码，`println!` 用来生成打印逻辑。
- `"{}"` 是格式化字符串，里面的 `{}` 是占位符。
- `println!("{}", other);` 表示把 `other` 的值填进 `{}` 的位置，然后打印出来。
- 这不是本次 ownership 的核心目标，后续示例应减少格式化负担，必要时先讲 `println!` 基础。
- Rust 基础类型可先分为：整数、浮点数、布尔值、字符、字符串相关类型、元组、数组。当前 ownership 学习只需要重点区分简单值如 `i32` 和拥有堆数据的 `String`。
- 不是只有 `String` 会发生所有权转移。更准确的规则是：实现了 `Copy` 的类型赋值时会复制，未实现 `Copy` 的类型赋值时通常会 move。数字、`bool`、`char` 多数是 `Copy`；`String`、`Vec<T>` 等拥有堆数据的类型通常不是 `Copy`。
- `&str` 是引用类型，引用本身是 `Copy`。复制 `&str` 时复制的是“指向字符串数据的位置和长度”，不是复制字符串内容本身。因此多个 `&str` 可以指向同一段字符串数据。
- 重新给 `b` 赋值不会改变 `a`。但 Rust 中变量默认不可变，必须写 `let mut b = a;` 才能执行 `b = ...`。`&str` 的例子里，`b = "world"` 只是让 `b` 指向另一段字符串字面量，`a` 仍指向 `"hello"`；整数例子里，`b = 2` 也不会影响 `a`，`a` 仍为 `1`。
- Rust 可以通过可变引用 `&mut` 修改原变量，语法类似 `let r = &mut a; *r = 2;`。但 Rust 限制更严格：变量本身要声明为 `mut`，同一时间通常只能有一个可变引用，不能同时存在会冲突的普通引用和可变引用。
- `i32` 实现了 `Copy`，但 `Copy` 不是 `.copy()` 方法。`Copy` 的复制发生在赋值、传参、返回值等位置，例如 `let b = a;`。如果想显式调用方法，应使用 `Clone` 的 `.clone()`，整数也实现了 `Clone`，但对 `i32` 来说通常没必要。

后续融入方式：

- 下次基础示例中，先说明 `println!` 是打印宏，`{}` 是要替换成变量值的位置。
- 讲 ownership 时优先对比 `i32` 这类 Copy 类型和 `String` 这类会 move 的类型。
- 在进入借用前，补一个小节说明 `Copy` 和 move 的判断规则。
- 讲 `&T` 借用时，强调“引用本身可以 Copy”和“被引用的数据没有被复制”是两件事。
- 讲变量绑定时补充：`let` 默认不可变，重新赋值需要 `mut`，重新绑定一个变量不等于修改另一个变量。
- 后续进入借用章节时，用 `&mut` 对比 C 指针和 JS 对象引用：Rust 能改原值，但必须通过编译器可检查的独占可变借用完成。
- 讲 `Copy` 时明确：`Copy` 是标记类型可隐式复制的 trait，不提供 `.copy()` 方法；`.clone()` 才是显式复制方法。

### 2026-06-12：trait 里的 `impl Future + Send`

问题：

- `'static` 是什么意思？这是 Rust 的保留字吗？
- 有其他加 `'` 的写法吗？

回答摘要：

- `'static` 是 Rust 里一个特殊的生命周期名称，表示“这个引用或异步任务不依赖某个短暂作用域里的借用”。
- 它不是普通变量名；从语言使用角度看，它是内置的特殊生命周期标识。
- `&'static str` 通常表示这个字符串引用在整个程序运行期间都有效，例如字符串字面量。
- 在 `impl Future + Send + 'static` 里，`'static` 通常表示返回的 future 里面不能持有短生命周期引用，因此可以被安全地交给运行时保存、调度或跨线程执行。
- 其他常见写法包括生命周期参数 `'a`、引用生命周期 `&'a T`、函数泛型 `fn get<'a>(...) -> &'a T`、结构体字段 `struct User<'a> { name: &'a str }`、trait bound `T: 'a`，以及较少需要手写的 `'_` 匿名生命周期。

后续融入方式：

- 后续讲 async、线程和借用时，增加“future 不能借用临时局部变量后被长期保存”的例子。
- 后续讲引用和结构体时，对比 `&T`、`&'a T`、`&'static T` 的差异。

### 2026-06-07：`if let` 链 + `let ... else`

问题：

- `Some` 是什么？
- `match` 怎么返回值？
- `let ... else` 的 `else` 为什么必须退出？
- Rust 里不加分号是不是等于提前退出？
- 为什么不能直接写 `let Some(user) = maybe_user;`？

回答摘要：

- `Some` 是 `Option<T>` 的有值分支，`None` 表示没有值。
- `match` 是表达式，会返回命中分支的值，所有分支类型必须一致。
- `let ... else` 的 `else` 块必须让当前流程退出，例如 `return`、`break`、`continue`、`panic!`。
- 不加分号表示代码块返回该表达式的值，但不等于提前退出函数。
- `let Some(user) = maybe_user;` 不行，因为 `Some(user)` 不是一定能匹配成功。

后续融入方式：

- 讲解 `Option` / `Result` 时，继续用 `Some` / `None` 和提前返回做例子。

### 2026-06-08：`filter_map` 和迭代器链

问题：

- `into_iter()` 是什么？
- `|value| value` 是什么？
- 为什么 `Vec` 不能直接用 `filter_map`？

回答摘要：

- `into_iter()` 把集合变成迭代器，通常会消耗原集合。
- `|value| value` 是闭包，表示接收 `value` 并原样返回。
- `Vec` 是容器，`Iterator` 才能使用 `filter_map`、`map`、`filter`、`collect` 等流水线操作。

后续融入方式：

- 讲所有权时，重点解释 `into_iter()` 为什么会消耗集合。
- 讲借用时，对比 `iter()`、`iter_mut()`、`into_iter()`。
