1. limited preview.
A limited preview is an early release available to a restricted group of users.
OpenAI is beginning a limited preview of the GPT-5.6 series.
A limited preview can help teams gather feedback before a broader rollout.

2. flagship model.
A flagship model is the most advanced or representative model in a product family.
Sol is positioned as the flagship model in the GPT-5.6 series.
Flagship models usually prioritize capability over cost.

3. balanced model.
A balanced model offers a practical tradeoff between capability, speed, and cost.
Terra is described as a balanced model for everyday work.
A balanced model may be better than a flagship model for high-volume workflows.

4. max reasoning effort.
Max reasoning effort means the model uses more computation or reasoning depth for harder tasks.

Max reasoning effort is useful when correctness matters more than latency.

5. ultra mode.
Ultra mode is a higher-capability mode that can go beyond a single-agent workflow.

Teams should reserve ultra mode for tasks that justify the added cost.

6. subagents.
Subagents are smaller or parallel agents that help complete parts of a larger task.
Subagents can accelerate complex work by splitting it into parallel steps.
An orchestrator can assign subtasks to several subagents.

7. cybersecurity capability.
Cybersecurity capability is a model's ability to reason about, test, or assist with security tasks.

8. real-world adversarial pressure.
Real-world adversarial pressure means pressure from realistic attackers, misuse attempts, or hostile conditions.
Safeguards must hold up to real-world adversarial pressure.
Evaluations should account for real-world adversarial pressure rather than idealized tests.

9. legitimate defensive work.
Legitimate defensive work means authorized security work intended to protect systems.

10. exploitation primitives.
Exploitation primitives are basic techniques or building blocks that can be used in an exploit.
The model identified bugs and exploitation primitives under the tested conditions.
Recognizing exploitation primitives is not the same as producing a full-chain exploit.

11. functional full-chain exploit.
A functional full-chain exploit is a complete working exploit that chains multiple steps to achieve an attack.
The model did not autonomously produce a functional full-chain exploit.
Generating a functional full-chain exploit would raise serious safety concerns.

12. layered safeguard stack.
A layered safeguard stack is a set of multiple safety mechanisms that work together.
A layered safeguard stack combines refusals, checks, monitoring, enforcement, and continued testing.
Layered safeguards are more robust than a single refusal rule.

13. differentiated access.
Differentiated access means different users or contexts receive different levels of access.
Differentiated access can preserve legitimate use while limiting higher-risk behavior.
Enterprise deployments often require differentiated access controls.

14. automated red teaming.
Automated red teaming uses automated systems or models to find weaknesses in safeguards.
Automated red teaming can test safeguards against many prompts or contexts.
Teams use automated red teaming to improve safety before release.

15. ambient behavior.
Ambient behavior is background, context-aware behavior that continues without interrupting users.
Ambient behavior lets Claude proactively surface relevant information.
Good ambient behavior helps prevent important work from stalling.
