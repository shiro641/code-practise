# 010. 预测赢家

## 状态

已完成。

## 题目

给定一个整数数组 `nums`。两个玩家轮流从数组两端取一个数，每次只能取最左边或最右边的一个，直到数组为空。每个玩家都会采用最优策略使自己的总分最大。

请你判断先手玩家最终是否能赢。这里“赢”定义为：先手玩家得分大于或等于后手玩家得分时，返回 `true`，否则返回 `false`。

## 示例

```text
输入: nums = [1,5,2]
输出: false
```

```text
输入: nums = [1,5,233,7]
输出: true
```

## 限制

```text
1 <= nums.length <= 20
0 <= nums[i] <= 10^7
```

## 核心识别

这是 `区间 DP` 中带有对抗语义的博弈型题目。

- 每一步只能从区间两端取数，所以决策天然发生在子区间 `[i, j]` 上。
- 不能贪心地只看两端哪个更大，因为当前选择会影响下一轮留给对手的区间。
- 更合适的建模方式是直接记录“当前先手相对后手最多能领先多少分”。

## 你的解答

你给出的核心建模是：

```text
dp[i][j] 表示在 [i, j] 区间内，当前先手最终最多能比后手多多少分。
```

转移为：

```text
dp[i][j] = Math.max(nums[i] - dp[i + 1][j], nums[j] - dp[i][j - 1])
```

TypeScript 实现见 `solution.ts`。

## 评价

通过。

状态定义正确：`dp[i][j]` 直接描述了当前玩家在区间 `[i, j]` 先手时，相对对手的最优分差，这个定义既贴合题意，也让转移非常自然。

转移正确：

```ts
dp[i][j] = Math.max(nums[i] - dp[i + 1][j], nums[j] - dp[i][j - 1]);
```

含义是：

- 如果先手拿左端 `nums[i]`，之后对手会在 `[i + 1, j]` 这个区间先手，因此当前最终分差是 `nums[i] - dp[i + 1][j]`
- 如果先手拿右端 `nums[j]`，同理分差是 `nums[j] - dp[i][j - 1]`
- 两者取最大，就是当前先手能达到的最好结果

遍历顺序正确：外层 `i` 从大到小，内层 `j` 从小到大，保证 `dp[i + 1][j]` 和 `dp[i][j - 1]` 在计算 `dp[i][j]` 前已经就绪，符合区间 DP 的依赖方向。

边界条件正确：`dp[i][i] = nums[i]`，表示区间里只剩一个数时，当前先手直接拿走它，最终就会领先 `nums[i]` 分。

最终判定条件也已修正为：

```ts
return dp[0][nums.length - 1] >= 0;
```

这点很关键，因为题目把“平局也算先手赢”定义为 `true`。

复杂度：

```text
时间复杂度：O(n^2)
空间复杂度：O(n^2)
```
