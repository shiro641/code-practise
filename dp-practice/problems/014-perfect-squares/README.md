# 014. 完全平方数

## 状态

已完成。

## 题目

给你一个整数 `n`，返回和为 `n` 的完全平方数的最少数量。

完全平方数是一个整数乘以它自己得到的数，比如 `1、4、9、16` 都是完全平方数。

## 示例

```text
示例 1：
输入：n = 12
输出：3
解释：12 = 4 + 4 + 4
```

```text
示例 2：
输入：n = 13
输出：2
解释：13 = 4 + 9
```

## 限制

```text
1 <= n <= 10^4
```

## 核心识别

这是 `完全背包` 的最值型问题。

- 可选物品是所有不超过 `n` 的完全平方数：`1, 4, 9, 16, ...`。
- 每个完全平方数都可以重复使用，所以是完全背包。
- 目标不是求方案数，而是求凑出和为 `n` 所需的最少个数，所以是最值型。

## 你的解答

你提交的是 Python 实现：

```python
nub_ = 13


def getMinCount(nub):
    
    items = [x * x for x in range(nub) if x * x <= nub]
    print(items)

    dp = [float('inf')] * (nub+1)
    dp[0] = 0

    print(dp)

    for i in range(len(items)):
        for j in range(items[i], nub+1):
            dp[j] = min(dp[j-items[i]] + 1, dp[j])

    print(dp)
    return dp[nub]

if __name__ == "__main__":
    print(getMinCount(nub_))
```

## 评价

通过。

虽然你这次还是没有先把状态定义和转移单独写出来，但从代码可以反推出你的建模是对的：

- `dp[j]` 表示凑出和为 `j` 所需的最少完全平方数个数。
- `dp[j] = min(dp[j], dp[j - items[i]] + 1)` 表示当前考虑完全平方数 `items[i]` 时，尝试再使用一个它来更新最优值。

遍历顺序正确：

- 外层遍历物品，也就是每个完全平方数。
- 内层 `j` 从小到大正序遍历。
- 这里正序很关键，因为同一个完全平方数允许被重复使用；如果倒序，就会错误地变成 0-1 背包。

边界条件正确：

- `dp[0] = 0`，表示凑出和为 0 不需要任何数。
- 其余位置初始化为 `inf`，表示初始不可达。

复杂度正确：

```text
时间复杂度：O(n * sqrt(n))
空间复杂度：O(n)
```

这份代码里有一个小问题，但不影响最终正确性：

- `items` 里把 `0` 也放进去了，因为你用了 `range(nub)`，所以第一项会是 `0 * 0 = 0`。
- 这个物品没有实际意义，虽然不会把结果算错，但属于无效物品，应该去掉。
- 更干净的写法是从 `1` 开始枚举平方数。

例如：

```python
items = [x * x for x in range(1, int(nub ** 0.5) + 1)]
```
