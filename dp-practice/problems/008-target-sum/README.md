# 008. 目标和

## 状态

已完成。

## 题目

给你一个非负整数数组 `nums` 和一个整数 `target`。

你可以给数组中的每个整数前添加 `'+'` 或 `'-'`，然后串联起所有整数，构造一个表达式。

返回可以通过这种方法构造出的、运算结果等于 `target` 的不同表达式的数目。

## 示例

```text
输入：nums = [1,1,1,1,1], target = 3
输出：5
```

```text
输入：nums = [1], target = 1
输出：1
```

## 限制

```text
1 <= nums.length <= 20
0 <= nums[i] <= 1000
0 <= sum(nums[i]) <= 1000
-1000 <= target <= 1000
```

## 核心识别

这是 `0-1 背包` 变体，目标是求方案数。

- 给每个数选择 `+` 或 `-`，本质上等价于把部分数选进一个子集。
- 设选为正号的子集和为 `positive`，负号子集和为 `negative`：

```text
positive - negative = target
positive + negative = sum
=> positive = (sum + target) / 2
```

- 因此问题转化为：从数组中选若干个数，使它们的和恰好等于 `bagSize = (sum + target) / 2`，求方案数。
- 每个数只能选或不选一次，所以是 `0-1 背包`，不是完全背包。

见 `solution.ts`。

补充说明：

```text
dp[j] 表示：从前 i 个数里选若干个数，凑出和为 j 的方案数。
```

## 评价

通过。

状态定义正确：一维压缩后，`dp[j]` 表示当前处理到前若干个数时，凑出和为 `j` 的方案数。

转移正确：

```ts
dp[j] = dp[j] + dp[j - nums[i]];
```

含义是：不选当前数的方案数，加上选当前数的方案数。

遍历顺序正确：外层遍历物品，内层 `j` 从大到小倒序遍历，保证每个数只被使用一次，符合 `0-1 背包`。

边界条件正确：

- `Math.abs(target) > sum` 时不可能拼出目标值，直接返回 `0`。
- `(sum + target) % 2 !== 0` 时无法转成整数容量，直接返回 `0`。
- `dp[0] = 1` 表示“什么都不选”这一种方案。

复杂度：

```text
时间复杂度：O(n * bagSize)
空间复杂度：O(bagSize)
```
