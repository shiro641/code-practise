# 013. 买卖股票的最佳时机含手续费

## 状态

已完成。

## 题目

给定一个整数数组 `prices`，其中 `prices[i]` 表示第 `i` 天股票的价格；另给定一个整数 `fee`，表示交易股票的手续费用。

你可以无限次地完成交易，但是每笔交易都需要付手续费。你不能同时参与多笔交易，也就是说你在再次购买前必须先卖出掉之前持有的股票。

返回获得利润的最大值。

## 示例

```text
示例 1：
输入：prices = [1,3,2,8,4,9], fee = 2
输出：8
```

```text
示例 2：
输入：prices = [1,3,7,5,10,3], fee = 3
输出：6
```

## 限制

```text
1 <= prices.length <= 5 * 10^4
1 <= prices[i] < 5 * 10^4
0 <= fee < 5 * 10^4
```

## 核心识别

这是 `股票状态机 DP` 的两状态模型。

- 每天结束时只关心两种状态：`不持股` 和 `持股`。
- 因为允许无限次交易，所以不需要额外记录交易次数维度。
- 与“买卖股票的最佳时机 II”相比，核心区别是每次完整交易都要多付一个 `fee`。

## 你的解答

你提交的是 Python 实现：

```python
prices_ = [1,3,2,8,4,9]
fee_ = 2


def getMaxMoney(prices, fee):
    dp = [[0] * 2 for _ in range(len(prices))]

    dp[0][0] = 0
    dp[0][1] = -prices[0]

    for i in range(1, len(prices)):
        dp[i][0] = max(dp[i - 1][1] - fee + prices[i], dp[i - 1][0])
        dp[i][1] = max(dp[i - 1][0] - prices[i], dp[i - 1][1])

    print(dp)
    return dp[len(prices)-1][0]


if __name__ == "__main__":
    print(getMaxMoney(prices_, fee_))
```

## 评价

通过。

从你的代码可以反推出状态定义是正确的：

- `dp[i][0]` 表示第 `i` 天结束后，手里 `不持有股票` 时的最大利润。
- `dp[i][1]` 表示第 `i` 天结束后，手里 `持有股票` 时的最大利润。

你的转移也正确：

```text
dp[i][0] = max(dp[i - 1][0], dp[i - 1][1] + prices[i] - fee)
dp[i][1] = max(dp[i - 1][1], dp[i - 1][0] - prices[i])
```

含义分别是：

- 今天结束时不持股：要么昨天就不持股，今天不操作；要么昨天持股，今天卖出，并在卖出时扣手续费。
- 今天结束时持股：要么昨天就持股，今天不操作；要么昨天不持股，今天买入。

遍历顺序正确：

- 从第 1 天顺序遍历到最后一天。
- 因为 `dp[i]` 只依赖 `dp[i - 1]`，所以按天从前往后推即可。

边界条件正确：

- `dp[0][0] = 0`
- `dp[0][1] = -prices[0]`

复杂度正确：

```text
时间复杂度：O(n)
空间复杂度：O(n)
```

这题还有两个你要继续压实的表达点：

1. 手续费放在“卖出时扣”或“买入时扣”都可以，本质上只是把同一笔交易成本记在不同一步里，只要整套转移前后一致即可。
2. 和“买卖股票的最佳时机 II”相比，这题唯一核心变化就是完整交易要多扣一个 `fee`；状态数量没有变，买入来源也没有变。
