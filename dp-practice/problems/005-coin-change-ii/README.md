# 005. 零钱兑换 II

## 状态

已完成。

## 题目

给你一个整数数组 `coins`，表示不同面额的硬币；再给你一个整数 `amount`，表示总金额。

请你计算并返回可以凑成总金额的硬币组合数。

如果任何硬币组合都无法凑出总金额，返回 `0`。

每种面额的硬币有无限个。

注意：顺序不同但组成相同的组合，算同一种组合。

## 示例

```text
输入：amount = 5, coins = [1,2,5]
输出：4
解释：
5=5
5=2+2+1
5=2+1+1+1
5=1+1+1+1+1
```

```text
输入：amount = 3, coins = [2]
输出：0
```

```text
输入：amount = 10, coins = [10]
输出：1
```

## 限制

```text
1 <= coins.length <= 300
1 <= coins[i] <= 5000
coins 中所有值互不相同
0 <= amount <= 5000
```

## 核心识别

这是完全背包计数组合题。

- 每种硬币可以无限次使用，所以不是 0-1 背包。
- 题目问组合数，顺序不同但组成相同只算一种。
- 外层遍历硬币，内层正序遍历金额，可以避免把排列重复计入。

## 你的解答

见 `solution.ts`。

补充说明：

```text
状态转移：决策容量 j 的背包有多少种装法：装 i，j - coins[i]；不装 i，j 本身，所以是这两种之和。
完全背包：因为数字可以重复用。
正序：因为完全背包，正序遍历前序状态可复用。
```

## 评价

通过。

状态定义基本正确：`dp[j]` 表示在当前已处理的硬币种类范围内，凑出金额 `j` 的组合数。

转移正确：

```ts
dp[j] += dp[j - coin]
```

含义是：如果最后再放入一个面额为 `coin` 的硬币，那么前面需要先凑出 `j - coin`。

遍历顺序正确：外层遍历硬币，内层金额正序遍历。正序允许同一种硬币在当前轮被重复使用，符合完全背包；外层固定硬币种类顺序，保证统计组合数，不把不同排列重复统计。

边界条件正确：`dp[0] = 1`，表示凑出金额 0 有一种方式，即什么都不选。

复杂度：

```text
时间复杂度：O(coins.length * amount)
空间复杂度：O(amount)
```

后续要继续强化的一点：说明组合数时，要主动提到“外层硬币、内层金额”如何避免排列重复。
