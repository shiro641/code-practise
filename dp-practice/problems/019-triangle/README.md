# 019. 三角形最小路径和

## 题目

给定一个三角形 `triangle` ，找出自顶向下的最小路径和。

每一步只能移动到下一行中相邻的结点上。也就是说，如果当前在第 `i` 行第 `j` 个位置，那么下一步只能走到第 `i + 1` 行的第 `j` 个或第 `j + 1` 个位置。

## 示例

```text
示例 1：
输入：triangle = [[2],[3,4],[6,5,7],[4,1,8,3]]
输出：11
解释：自顶向下的最小路径和为 2 + 3 + 5 + 1 = 11
```

```text
示例 2：
输入：triangle = [[-10]]
输出：-10
```

## 限制

```text
1 <= triangle.length <= 200
triangle[0].length == 1
triangle[i].length == triangle[i - 1].length + 1
-10^4 <= triangle[i][j] <= 10^4
```

## 用户解答

你提交的是 Python 实现：

```python
input = [[2], [3, 4], [6, 5, 7], [4, 1, 8, 3]]


def getMinPath(input):
    dp = [[float("inf")] * len(input[-1]) for _ in input]

    dp[0][0] = input[0][0]

    for i in range(1, len(input)):
        for j in range(len(input[i])):
            dp[i][j] = (
                dp[i - 1][j] if j == 0 else min(dp[i - 1][j], dp[i - 1][j - 1])
            ) + input[i][j]

    print(dp)
    return min(dp[-1])


if __name__ == "__main__":
    print(getMinPath(input))
```

## 评价

通过。

这份代码的核心建模是正确的：

- `dp[i][j]` 表示从顶点走到第 `i` 行第 `j` 个位置时的最小路径和。
- 状态转移本质上是从上一行相邻位置转移：
  - 左边界 `j == 0` 时，只能从 `dp[i - 1][j]` 转移。
  - 右边界 `j == i` 时，只能从 `dp[i - 1][j - 1]` 转移。
  - 中间位置可从 `min(dp[i - 1][j], dp[i - 1][j - 1])` 转移。

你最后使用了“矩形数组 + `inf` 哨兵”的写法来省掉显式右边界分支，这个技巧是成立的。它的关键前提是：

- 逻辑上你仍然知道合法来源只有上一行的相邻位置。
- 非法来源之所以没有把结果算错，是因为对应位置被初始化为 `inf`，不会被 `min(...)` 选中。

遍历顺序正确：

- 外层按行从上到下遍历。
- 内层按当前行从左到右遍历。
- 由于每个状态只依赖上一行，因此这样遍历可以保证依赖已就绪。

边界条件正确：

- `dp[0][0] = triangle[0][0]`
- 其余位置先初始化为 `inf`，表示默认不可达，再由合法转移更新。

复杂度正确：

```text
时间复杂度：O(n^2)
空间复杂度：O(n^2)
```

其中 `n` 为三角形的行数。
