# 011. 零钱兑换

## 题目

给你一个整数数组 `coins`，表示不同面额的硬币；再给你一个整数 `amount`，表示总金额。

你可以认为每种硬币的数量是无限的。返回可以凑成总金额所需的最少硬币个数。如果任何一种硬币组合都无法凑出总金额，返回 `-1`。

## 示例

```text
示例 1：
输入：coins = [1, 2, 5], amount = 11
输出：3
解释：11 = 5 + 5 + 1
```

```text
示例 2：
输入：coins = [2], amount = 3
输出：-1
```

```text
示例 3：
输入：coins = [1], amount = 0
输出：0
```

## 限制

```text
1 <= coins.length <= 12
1 <= coins[i] <= 2^31 - 1
0 <= amount <= 10^4
```

## 核心识别

这是 `完全背包` 的最值型问题，不是计数型问题。

- 每种硬币都可以重复使用，所以同一种物品可以选多次。
- 目标不是求方案数，而是求“凑出金额 `j` 的最少硬币数”。
- 和你之前做过的 `零钱兑换 II` 区别在于：那题的状态语义是“方案数”，这题的状态语义是“最少个数”。

## 你的解答

你提交的是 Python 实现：

```python
coins_ = [1, 2, 5]
amount_ = 11


def getMinNum(coins, amount):
    UN_REACH = 999999
    dp = [UN_REACH] * (amount + 1)
    dp[0] = 0
    for i in range(0, len(coins)):
        for j in range(coins[i], amount + 1):
            dp[j] = min(dp[j], dp[j - coins[i]] + 1)
    print(-1 if dp[amount] == UN_REACH else dp[amount])


if __name__ == "__main__":
    getMinNum(coins_, amount_)
```

## 评价

通过。

虽然你这次没有先把状态定义和转移单独写出来，但从代码可以反推出你的建模是对的：

- `dp[j]` 表示凑出金额 `j` 所需的最少硬币数。
- `dp[j] = min(dp[j], dp[j - coins[i]] + 1)` 表示当前考虑硬币 `coins[i]` 时，如果能从金额 `j - coins[i]` 转移过来，就尝试用一枚当前硬币更新最优值。

遍历顺序正确：

- 外层遍历硬币，内层 `j` 从小到大正序遍历。
- 这里正序很关键，因为完全背包允许同一种硬币在同一轮内被重复使用；如果倒序，就会错误地限制每种硬币只能用一次，变成 0-1 背包。

边界条件正确：

- `dp[0] = 0`，表示凑出金额 0 需要 0 枚硬币。
- 其余位置初始化为一个不可达的大数，表示初始时这些金额都还凑不出来。

复杂度正确：

```text
时间复杂度：O(n * amount)
空间复杂度：O(amount)
```
