# 007. 不同路径 II

## 状态

已完成。

## 题目

给定一个 `m x n` 的网格 `obstacleGrid`。机器人初始位于左上角 `obstacleGrid[0][0]`，每次只能向下或者向右移动一步。

网格中的 `1` 表示障碍物，`0` 表示空位置。请返回机器人到达右下角 `obstacleGrid[m - 1][n - 1]` 的不同路径数量。

## 示例

```text
输入：
obstacleGrid = [
  [0,0,0],
  [0,1,0],
  [0,0,0]
]
输出：2
```

```text
输入：
obstacleGrid = [
  [0,1],
  [0,0]
]
输出：1
```

## 限制

```text
1 <= m, n <= 100
obstacleGrid[i][j] 只可能是 0 或 1
```

## 核心识别

这是二维路径 / 网格 DP。

- 每个位置的路径数只依赖它上方和左方的位置。
- 障碍物不能经过，因此该位置路径数为 `0`。
- 第一行和第一列由于只能单方向到达，需要单独初始化。

## 你的解答

见 `solution.ts`。

补充说明：

```text
1. dp[i][j]，到达 i，j 有多少种路径
2. 第一行和第一列遇到障碍就设置当前路径为 0，由于后面状态依赖前面的，所以后续也会保持为 0
```

## 评价

通过。

状态定义正确：`dp[i][j]` 表示从左上角走到位置 `(i, j)` 的不同路径数量。

转移正确：

```ts
if (obstacleGrid[i][j] === 1) {
  dp[i][j] = 0;
} else {
  dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
}
```

遍历顺序正确：先初始化起点、第一列、第一行，再从上到下、从左到右填完整个二维表。

边界条件正确：

- 起点是障碍物时，答案应为 `0`。
- 第一列一旦遇到障碍，后面位置都无法仅靠向下到达，因此会继承为 `0`。
- 第一行同理，一旦遇到障碍，后面位置都无法仅靠向右到达，因此会继承为 `0`。
- 障碍物位置自身的 `dp` 值始终为 `0`。

复杂度：

```text
时间复杂度：O(m * n)
空间复杂度：O(m * n)
```
