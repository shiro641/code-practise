关键是这题问的是 最长回文子序列，不是 最长回文子串。

对于：

s = "bbbab"

可以选这些字符：

b b b b

也就是：

第 0 个 b
第 1 个 b
第 2 个 b
第 4 个 b

组成：

"bbbb"

它是回文，长度是 4。

如果是 最长回文子串，连续的一段最长是：

"bbb"

长度才是 3。

所以区别是：

子串：必须连续
子序列：可以不连续，但相对顺序不能变

题目：最长回文子序列

给你一个字符串 s。

请你找出其中最长的回文子序列，并返回该序列的长度。

示例 1：

输入：
s = "bbbab"

输出：
4

示例 2：

输入：
s = "cbbd"

输出：
2

示例 3：

输入：
s = "abcde"

输出：
1

限制：

1 <= s.length <= 1000

s 仅由小写英文字母组成

注意昨天已经确认过：

"bbbab" 的答案是 4

因为题目问的是 子序列，不是 子串。子序列可以不连续。

const s = "bbbab"
const getHuiStr = (s: string) => {
    const dp = Array.from({ length: s.length }, (_, i) => Array.from({ length: s.length }, (_, j) => i === j ? 1 : 0));
    // 倒着遍历，因为j很快就遍历到底了，所以可以先算出小区间的状态
    for (let i = s.length - 1; i >= 0; i--) {
        for (let j = i + 1; j < s.length; j++) {
            if (s[i] === s[j]) {
                dp[i][j] = i + 1 > j - 1 ? 2 : dp[i + 1][j - 1] + 2
            } else {
                dp[i][j] = Math.max(dp[i + 1][j], dp[i][j - 1])
            }
        }
    }
    // console.log(dp, )
    return dp[0][s.length-1]
}

getHuiStr(s)

这版是对的 ✅

你的核心状态和遍历顺序都没问题：

dp[i][j]: s[i...j] 区间内的最长回文子序列长度

转移也正确：

if (s[i] === s[j]) {
  dp[i][j] = dp[i + 1][j - 1] + 2
} else {
  dp[i][j] = Math.max(dp[i + 1][j], dp[i][j - 1])
}

你额外写的这个判断：

i + 1 > j - 1 ? 2 : dp[i + 1][j - 1] + 2

也没问题，用来处理长度为 2 的区间，比如 "bb"。

不过其实可以简化，因为 dp[i + 1][j - 1] 在这种场景下可以视为 0，但 JS 数组越界会拿到 undefined，所以你这样写更安全。
